# GSQ-006: Queue Section (Split-View links)

## User Story
Als Developer möchte ich die Queue im Bottom Panel sehen und verwalten, damit ich Specs zur Ausführung einreihen, umordnen und starten/stoppen kann.

## Akzeptanzkriterien
- GEGEBEN das Queue & Specs Tab
  DANN ist links die Queue-Section mit Start/Stop Controls sichtbar

- GEGEBEN Specs in der Queue
  WENN ich ein Item per Drag & Drop verschiebe
  DANN ändert sich die Reihenfolge

- GEGEBEN eine Spec-Card wird vom Specs-Bereich (rechts) auf die Queue gedraggt
  DANN wird die Spec zur Queue hinzugefügt

- GEGEBEN eine laufende Queue
  DANN zeigt jedes Item den aktuellen Fortschritt (x/y Stories) und den Projektnamen

## Technische Details

**Type:** Frontend
**Priority:** Critical
**Effort:** M (5 SP)
**Abhängigkeiten:** GSQ-005

### WAS
- Neue Komponente `aos-queue-section.ts`
- Logik migriert aus `aos-queue-sidebar.ts` (~70%)

### WIE
- Presentation Layer: Light DOM Subkomponente des Panels
- Wiederverwendbare Logik aus `aos-queue-sidebar.ts`:
  - boundMessageHandlers Map Pattern
  - Event Dispatching (queue-add, -remove, -reorder, -start, -stop)
  - Git Strategy Dialog Integration
  - Progress Tracking (Map-basiert pro specId)
  - Drag-Drop Reordering mit Position-Targeting
- Neues Layout: Vertikal scrollbare Liste
  - Header: Start/Stop Buttons, Item-Count, Running-Indicator
  - Items: Kompakte Cards mit Projekt-Badge, Progress-Bar, Status-Text
- Drop-Zone: Akzeptiert `text/drag-type === 'spec'` mit visuellen Highlight
- dataTransfer liest: specId, specName, projectPath, gitStrategy
