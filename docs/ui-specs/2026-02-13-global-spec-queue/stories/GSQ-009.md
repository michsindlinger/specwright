# GSQ-009: Dashboard Queue-Sidebar entfernen

## User Story
Als Developer möchte ich, dass die alte Queue-Sidebar aus dem Dashboard entfernt wird, damit die Queue nur noch global im Bottom Panel existiert und das Dashboard mehr Platz hat.

## Akzeptanzkriterien
- GEGEBEN das Dashboard
  DANN ist keine Queue-Sidebar mehr sichtbar

- GEGEBEN das Dashboard
  DANN nutzt die Spec-/Kanban-Ansicht die volle Breite

- GEGEBEN Spec-Cards im Dashboard
  DANN sind sie weiterhin draggable (für Drop in Bottom Panel)

- GEGEBEN die alte aos-queue-sidebar Komponente
  DANN ist sie entfernt (keine toten Code-Referenzen)

## Technische Details

**Type:** Full-Stack
**Priority:** High
**Effort:** S (3 SP)
**Abhängigkeiten:** GSQ-006, GSQ-007

### WAS
- `dashboard-view.ts`: Queue-Sidebar und alle Queue-Handler entfernen
- `aos-queue-sidebar.ts`: Datei entfernen
- `aos-queue-item.ts`: Prüfen ob noch benötigt, ggf. entfernen
- Spec-Cards: dataTransfer um `text/project-path` erweitern

### WIE
- Presentation Layer: Dashboard Cleanup
  - Entferne `aos-queue-sidebar` Import und Referenz
  - Entferne Queue-State Properties (queue, isQueueRunning)
  - Entferne alle Queue Event-Handler (handleQueueAdd/Remove/Reorder/Start/Stop)
  - Entferne Gateway Queue-Message Handler Registrierungen
  - Ersetze `dashboard-content-with-queue` Layout durch `dashboard-content` (volle Breite)
- spec-card.ts: Erweitere dataTransfer um `text/project-path` (aktuelles Projekt aus Context)
- Lösche: `components/queue/aos-queue-sidebar.ts`
- Prüfe: `components/queue/aos-queue-item.ts` - wenn in neuer Queue-Section nicht wiederverwendet, löschen

### WO
- `agent-os-ui/ui/src/views/dashboard-view.ts` (modify)
- `agent-os-ui/ui/src/components/queue/aos-queue-sidebar.ts` (delete)
- `agent-os-ui/ui/src/components/queue/aos-queue-item.ts` (delete or keep)
- `agent-os-ui/ui/src/components/spec-card.ts` (modify)

### Geschätzte Komplexität: S
