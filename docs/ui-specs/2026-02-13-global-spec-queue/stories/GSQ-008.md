# GSQ-008: Execution Log Tab

## User Story
Als Developer möchte ich ein Execution Log der Queue-Ausführung sehen, damit ich den Fortschritt und Fehler in Echtzeit nachverfolgen kann.

## Akzeptanzkriterien
- GEGEBEN eine laufende Queue
  WENN ein Spec/Story gestartet oder abgeschlossen wird
  DANN erscheint ein neuer Log-Eintrag in Echtzeit

- GEGEBEN das Log scrollt
  WENN neue Einträge kommen
  DANN scrollt das Log automatisch nach unten

- GEGEBEN der User hat manuell hochgescrollt
  WENN neue Einträge kommen
  DANN scrollt das Log NICHT automatisch (Scroll-Lock)

- GEGEBEN verschiedene Log-Typen
  DANN sind sie farbcodiert (blau=Start, grün=Complete, rot=Error)

## Technische Details

**Type:** Frontend
**Priority:** High
**Effort:** S (3 SP)
**Abhängigkeiten:** GSQ-005

### WAS
- Neue Komponente `aos-execution-log-tab.ts`

### WIE
- Presentation Layer: Light DOM Subkomponente des Panels
- Initial-Load: Gateway `queue.log.state` anfordern, Response enthält alle bisherigen Einträge
- Echtzeit: Gateway Handler für `queue.log.entry` - appended neue Einträge
- Darstellung: Scrollbare Liste mit monospace Font
  - Pro Eintrag: `[HH:MM:SS] [Projekt] [Spec] [Story?] Message`
  - Farbcodierung: CSS-Klassen per log-type (spec-start=blau, story-complete=grün, error=rot, queue-complete=gold)
- Auto-Scroll: scrollTop = scrollHeight nach neuem Eintrag
  - Deaktiviert wenn User manuell scrollt (scroll-Event Listener, Abstand zum Ende > Threshold)
  - Re-aktiviert wenn User ganz nach unten scrollt
- "Clear Log" Button: Leert lokale Anzeige (optional: Backend-Clear)

### WO
- `agent-os-ui/ui/src/components/queue/aos-execution-log-tab.ts` (new)

### Geschätzte Komplexität: S
