# Definition of Ready (DoR)

> Project: Agent OS Web UI
> Last Updated: 2026-01-30

## Overview

A story is considered READY for implementation when all of the following criteria are met.

---

## Functional Requirements

- [ ] User story clearly written (Als... möchte ich... damit...)
- [ ] User role identified (Developer, Power User)
- [ ] Business value described
- [ ] Acceptance criteria defined (2-5 Gherkin scenarios)

---

## Technical Requirements

- [ ] Story type identified (Backend, Frontend, Full-Stack, DevOps)
- [ ] WAS section complete (components to create/modify)
- [ ] WIE section complete (architecture patterns to follow)
- [ ] WO section complete (file paths listed)
- [ ] Geschätzte Komplexität assigned (XS, S, M)

---

## Dependencies

- [ ] Dependencies identified and listed
- [ ] Blocking stories completed OR no blockers
- [ ] Required APIs/services available

---

## Sizing

- [ ] Story size is appropriate (max 5 files, 400 LOC)
- [ ] Can be completed in reasonable time
- [ ] Not multiple features in one story

---

## Clarity

- [ ] No ambiguous requirements
- [ ] Edge cases identified
- [ ] Error scenarios described
- [ ] Questions answered before implementation

---

## Checklist for Story Authors

When creating a story, verify:
1. Feature description is specific and actionable
2. Acceptance criteria are testable
3. Technical approach is defined but not prescriptive
4. Story is appropriately sized
5. All dependencies are listed

---

## Ready Indicators

A story is ready when:
- All checkboxes above can be checked
- Team has no blocking questions
- Required context documents exist
- Story can be implemented independently
