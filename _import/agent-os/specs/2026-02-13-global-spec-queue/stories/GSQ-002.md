# GSQ-002: Multi-Project Spec-Loading

## User Story
Als Developer möchte ich Spezifikationen aus allen geöffneten Projekten sehen können, damit ich einen projektübergreifenden Überblick über alle verfügbaren Specs habe.

## Akzeptanzkriterien
- GEGEBEN mehrere geöffnete Projekte mit Specs
  WENN das Frontend `specs.list-all` anfragt
  DANN erhalte ich alle Specs gruppiert nach Projekt

- GEGEBEN ein Projekt ohne Specs
  WENN `specs.list-all` abgefragt wird
  DANN erscheint das Projekt mit leerer Spec-Liste

- GEGEBEN die Response von `specs.list-all`
  DANN enthält jede SpecInfo projectPath und projectName

## Technische Details

**Type:** Backend
**Priority:** Critical
**Effort:** S (3 SP)
**Abhängigkeiten:** keine

### WAS
- `specs-reader.ts`: Neue Methode `listAllSpecs(projectPaths: string[])`
- Neues Interface `ProjectSpecs`
- `SpecInfo` erweitern um `projectPath` und `projectName`
- Neuer WebSocket-Handler `specs.list-all`

### WIE
- Integration Layer: SpecsReader bekommt neue Methode
- Iteriert über alle projectPaths, ruft bestehende `listSpecs()` pro Projekt auf
- Reichert SpecInfo mit projectPath/projectName an
- WebSocket Handler registriert auf `specs.list-all`
- Response-Format: `{ projects: ProjectSpecs[] }`

### WO
- `agent-os-ui/src/server/specs-reader.ts` (modify)
- `agent-os-ui/src/server/handlers/` (new handler or extend existing)

### Geschätzte Komplexität: S
