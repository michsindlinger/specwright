# GSQ-007: Specs Section (Split-View rechts)

## User Story
Als Developer möchte ich alle Spezifikationen aus allen Projekten im Bottom Panel sehen, damit ich schnell Specs finden und in die Queue ziehen kann.

## Akzeptanzkriterien
- GEGEBEN mehrere offene Projekte mit Specs
  DANN zeigt die Specs-Section alle Specs gruppiert nach Projekt

- GEGEBEN eine Spec-Card in der Specs-Section
  WENN ich sie in die Queue-Section (links) ziehe
  DANN wird die Spec zur Queue hinzugefügt

- GEGEBEN eine Spec-Card
  WENN ich den [+Q] Button klicke
  DANN wird die Spec zur Queue hinzugefügt

- GEGEBEN ein neues Projekt wird geöffnet
  DANN aktualisiert sich die Specs-Section automatisch

## Technische Details

**Type:** Frontend
**Priority:** Critical
**Effort:** M (5 SP)
**Abhängigkeiten:** GSQ-005

### WAS
- Neue Komponente `aos-specs-section.ts`
- Compact Spec-Cards mit Drag-Fähigkeit

### WIE
- Presentation Layer: Light DOM Subkomponente des Panels
- Daten laden über `gateway.send({ type: 'specs.list-all' })`
- Handler für Response: `specs.list-all.response`
- Darstellung: Collapsible Projekt-Sektionen
  - Projekt-Header: Name + Spec-Count, klickbar zum Auf-/Zuklappen
  - Spec-Cards: Name, Status-Badge (ready/running/done/etc), Story-Progress (x/y), Git-Strategy Badge
- Draggable: jede Spec-Card hat `draggable="true"`
  - `dragstart` setzt: text/plain (specId), text/spec-name, text/drag-type (spec), text/project-path, text/git-strategy
  - Drag-Ghost zeigt Spec-Name + Projekt
- [+Q] Button: Dispatcht `queue-add` Event mit specId, specName, projectPath
- Git-Strategy Dialog: Wenn Spec keine Strategy hat, Dialog öffnen vor Queue-Add
- Auto-Refresh: Reagiert auf project-opened/project-closed Events, re-fetcht Specs

### WO
- `agent-os-ui/ui/src/components/queue/aos-specs-section.ts` (new)

### Geschätzte Komplexität: M
